# Terminal Prototyper

Сборщик интерактивных прототипов (как в Figma) для тестирования на терминалах под Android/Chrome,
с записью кликов и **тепловыми картами**.

- **Редактор** на ПК — экраны из картинок/видео, кликабельные зоны, переходы, свайпы, авто-переход по таймеру, автоплей видео.
- **Плеер (PWA)** — открывается в браузере терминала, играет во весь экран, пишет каждый тап.
- **Тепловые карты** — клики по экранам и сценариям, по сессиям и суммарно, с наложением сессий.
- Работает **и онлайн, и офлайн**: клики копятся на терминале (IndexedDB) и синхронизируются с сервером, когда есть сеть; плюс ручной экспорт/импорт результатов в `.json`.

## Запуск

```bash
npm install

# разработка (сервер :5174 + Vite :5173 с авто-перезагрузкой)
npm run dev

# боевой режим на одном порту (так открывают терминалы)
npm run build
npm start          # → http://<IP-этого-ПК>:5174
```

При старте сервер печатает в консоль адреса, в том числе для терминалов
(например `http://192.168.1.50:5174`).

## Как пользоваться

1. На ПК открой `http://localhost:5174` (или `:5173` в dev), создай прототип, задай размер холста
   (пропорции экрана терминала — портрет/квадрат/альбом).
2. Слева добавь экраны: **картинка** или **видео**. Картинки готовь под пропорции холста.
3. На картинке **зажми и потяни** — нарисуешь кликабельную зону. Справа задай действие
   (перейти на экран + анимация перехода / назад), подпись зоны, свайпы, таймер, поведение видео.
4. Подключи терминал к той же Wi-Fi, открой в Chrome адрес ПК, зайди в прототип → **Запустить**.
   Введи метку терминала/участника (необязательно) и жми «во весь экран».
5. После теста: **Тепловые карты** — выбери экран, режим (тепло / точки по сессиям),
   фильтр (все / по зонам / мимо), сессии для наложения.

### Меню оператора на терминале
Долгое нажатие (~0.7 c) в **левом верхнем углу** открывает меню: начать сначала,
новая сессия/участник, скачать результаты `.json`, выйти из полноэкранного.

## Офлайн на терминале

- Если терминал теряет сеть во время теста — клики не пропадают, копятся локально и
  досинхронизируются автоматически при возврате сети.
- Полностью офлайн-сценарий: на терминале «Скачать результаты (.json)», перенести файл на ПК и
  в разделе «Тепловые карты» → «Импорт результатов».
- Чтобы плеер работал офлайн и после перезагрузки страницы (service worker), нужен защищённый
  контекст. По локальной сети это HTTPS: положи `key.pem` и `cert.pem` в `data/certs/`
  (удобно сгенерировать через [mkcert](https://github.com/FiloSottile/mkcert):
  `mkcert <IP-ПК>`), и сервер дополнительно поднимется на HTTPS (порт 5175).
  Без этого онлайн-режим и сбор кликов всё равно работают.

## Данные

Всё лежит в папке `data/` рядом с проектом:
- `data/prototypes/*.json` — прототипы
- `data/events/*.ndjson` — логи кликов
- `data/uploads/*` — загруженные картинки и видео

Бэкап = скопировать папку `data/`.
