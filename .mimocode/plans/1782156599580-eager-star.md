# Редактирование зон нажатия после тестирования

## Суть
Сделать зоны нажатия интерактивными на тепловых картах: клик по зоне выделяет её и показывает статистику + поле редактирования названия в правой панели. Улучшить отображение статистики по зонам.

## Что сейчас есть
- `Hotspot.label?: string` — поле уже есть в типах, правая панель «Экран» уже позволяет его редактировать
- `HeatmapsView` уже считает `stats.byHotspot` (Map<hotspotId, count>) и показывает теги на зонах
- Теги показывают **либо** label **либо** `${count} кл.` — не вместе
- Зоны на карте имеют `pointer-events: none` — полностью неинтерактивны

## Изменения

### 1. `client/src/pages/editor/HeatmapsView.tsx`

**Добавить состояние выбора зоны:**
- `selHotspot: string | null` — ID выбранной зоны

**Сделать зоны кликабельными:**
- Убрать `pointer-events: none` с `.heat__hotspot`
- Добавить `onClick` → `setSelHotspot(h.id)` (с `stopPropagation`)
- Добавить класс `is-selected` когда зона выбрана

**Улучшить отображение тега зоны:**
- Показывать **и** label **и** количество кликов:
  - Если есть label: `Оплатить · 12 кл.`
  - Если нет label: `12 кл.`
- Это даёт полную информацию без переключения вкладок

**Добавить секцию «Зоны» в правую панель (между «Статистика» и экспорт-кнопками):**
- Список всех зон экрана с количеством кликов
- Каждая строка: название (или «Зона N») + badge с количеством
- Клик по строке → выбор зоны (выделяет её на canvas)
- Выбранная зона подсвечивается

**Добавить мини-редактор названия при выборе зоны:**
- Когда `selHotspot` установлен, показать над списком зон:
  - Заголовок «Зона»
  - `Input` с полем `label` (как в RightPanel экрана)
  - Бейдж с количеством кликов
  - Кнопка «Снять выделение»

**Передавать `selHotspot` и `onSelectHotspot` в HeatmapsView:**
- Из `Editor.tsx` — HeatmapsView уже получает `doc` и `selScreen`, добавить `selHotspot` и `onSelectHotspot`

### 2. `client/src/pages/Editor.tsx`

- Передать `selHotspot` и `setSelHotspot` в `<HeatmapsView>`

### 3. `client/src/styles/pages.css`

- `.heat__hotspot`: добавить `cursor: pointer`, убрать `pointer-events: none`
- `.heat__hotspot.is-selected`: стили выделения (аналогично `.hotspot.is-selected` в editor)
- `.heat__hotspot-tag`:調整 для отображения label + count

### 4. `client/src/export.ts` — улучшение презентации

Презентация **уже** показывает зоны на каждом слайде экрана (`zonesBlock`). Но сводный слайд показывает только статистику по экранам. Добавить:

**На каждый слайд экрана** — показать количество зон и топ-3 зоны с кликами (уже есть через `zonesBlock`, но убедиться что label отображается корректно)

**На сводный слайд** — добавить секцию «Топ зоны»: объединить зоны со всех экранов, показать топ-5 самых кликаемых зон с указанием экрана:
```
Топ зоны
Оплатить (Экран 1) ........... 47
Назад (Экран 2) .............. 23
...
```

- `client/src/pages/Editor.tsx` — прокидывание пропсов

## Верификация
1. `npm run dev` → открыть редактор → вкладка «Карты»
2. Зоны отображают и название, и количество кликов (например «Оплатить · 5 кл.»)
3. Клик по зоне → выделяется, в правой панели появляется поле редактирования названия
4. Ввод нового названия → тег на зоне обновляется
5. Клик по другой зоне → переключение выбора
6. Список зон в правой панели показывает все зоны с кликами
7. Экспорт презентации (HTML) → каждый слайд показывает зоны с кликами
8. Сводный слайд показывает топ зон across экранов
9. `npx tsc --noEmit` — без ошибок
