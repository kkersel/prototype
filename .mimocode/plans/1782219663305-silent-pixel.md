# Plan: Remove white backgrounds from screenshots in PDF presentation

## Problem
Screenshots in the PDF presentation have opaque rectangular backgrounds (visible as white areas around phone mockups where the user marked red crosses). This is caused by exporting canvas as JPEG (no transparency support).

## Root Cause
In `client/src/export.ts`, `renderScreenImage()` composites screenshots onto a canvas and exports as `canvas.toDataURL('image/jpeg', 0.85)`. JPEG doesn't support alpha transparency, so the rounded corners from CSS `border-radius:28px` reveal the opaque rectangle.

## Changes

### File: `client/src/export.ts`

1. **Line 108**: Change `canvas.toDataURL('image/jpeg', 0.85)` → `canvas.toDataURL('image/png')` in `renderHeat()`
2. **Line 139**: Change `canvas.toDataURL('image/jpeg', 0.85)` → `canvas.toDataURL('image/png')` in `renderDots()`
3. **Lines 92-93**: Remove `ctx.fillStyle = '#000'; ctx.fillRect(0, 0, w, h)` in `renderHeat()` — the canvas is transparent by default, no need for black fill
4. **Lines 117-118**: Remove `ctx.fillStyle = '#000'; ctx.fillRect(0, 0, w, h)` in `renderDots()` — same reason

That's it — 4 small edits in one file. The CSS `border-radius:28px` + `box-shadow` will then work correctly on the transparent PNG, showing rounded corners with shadows and no opaque rectangle.

## Verification

2. Export PDF via "Скачать PDF" button
