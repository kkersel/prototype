# Контекст продукта

## Цель
Собирать простые интерактивные прототипы (как в Figma) из картинок/видео и
воспроизводить их на платёжных терминалах, фиксируя каждый тап в тепловые карты.

## Целевая аудитория
Дизайнеры / продакты / UX-исследователи команды, тестирующие интерфейсы на терминалах.
Вторичная: операторы, проводящие сессии тестирования рядом с терминалом.

## Ценность
Реальное устройство + реальные касания + тепловые карты по этапам — то, чего не даёт
статичный Figma-прототип на десктопе. Работает и онлайн, и офлайн.

## Метрика успеха
Время от «есть картинки» до «прогнал сессию на терминале и увидел тепловую карту».

## В скоупе
- Редактор прототипов на ПК: экраны (картинка/видео), кликабельные зоны, переходы, свайпы, таймеры.
- Плеер для терминала (PWA, Chrome): фуллскрин, запись тапов, онлайн+офлайн.
- Тепловые карты: по экрану, по сессиям и суммарно, наложение сессий.

## Вне скоупа
- Сложная анимация/логика прототипа (условия, переменные, формы).
- Многопользовательское редактирование в реальном времени.
- Кросс-сценарное наложение тепловых карт разных прототипов (отложено).
- Нативное Android-приложение (решено делать веб).

## Сценарии

**S01 Сборка прототипа**
Кто: дизайнер на ПК.
Триггер: готовы картинки под размер экрана терминала.
Шаги: создать прототип → задать размер холста → добавить экраны → нарисовать зоны → связать переходы.
Результат: кликабельный прототип, сохранён на сервере.
Edge cases:
- картинка не совпадает по пропорциям с холстом → letterbox, координаты нормализуются;
- экран без медиа; зона без назначенного перехода (показываем как «не задан»).

**S02 Прогон на терминале**
Кто: оператор у терминала.
Триггер: терминал открыт на адресе ПК в Chrome.
Шаги: открыть прототип → «Запустить» → (опц.) метки участника/устройства → тест фуллскрин.
Результат: тапы записаны (нормализованные координаты, hit/miss, сессия).
Edge cases:
- сеть пропала во время теста → тапы копятся в IndexedDB, синк при возврате;
- фуллскрин недоступен/заблокирован → продолжаем без него.

**S03 Анализ тепловых карт**
Кто: исследователь на ПК.
Триггер: проведены сессии.
Шаги: открыть тепловые карты → выбрать экран → режим (тепло/точки) → фильтр → наложить сессии.
Результат: видно, куда жали и мимо ли зон.
Edge cases:
- нет данных по экрану → пустое состояние с подсказкой;
- офлайн-сессия на другом сервере → импорт результатов .json (идемпотентный merge).

> Готовность: цель + ЦА + ≥3 сценария — выполнено.
